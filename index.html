Correspondence

Author: Philip Dorrell

Correspondence is a project to interactively visualize correspondences between items in related structures.

Move your mouse over the structures below, and see what happens.

¿ Cuál prefieres ? Me gusta el verde .
Which one do you prefer ? I like the green one .
(More literally ...)
Which you prefer ? The green (one) pleases me .

(Hint: to de-select without selecting a new item, click anywhere not inside a selectable item.)

Explanation

Typically two structures might be related by translation. For example,

  • Translation between English and another natural language
  • Translation between English and a programming language
  • Translation between English and mathematical formulae
  • Translation between English and items in a diagram

(If you speak some language other than English, substitute your preferred language in the examples above.)

To see how the examples are coded, choose View Source in your web browser.

Abstract Example

The basic concepts of Correspondence are structure groups, structures and items.

The following example shows a structure group consisting of two structures, structure A and structure B, each containing three items. In structure B, the first item is split into two parts.

Structure A: Item 1, item 2 and item 3.
Structure B: This second structure contains its own representations of the three items: the first part of item 1, item 2, the second part of item 1 and item 3.

Some More Examples, Not Hand-Coded

All the examples above have been manually coded in HTML.

The HTML format is as simple and straightforward as I can make it, but it is quite laborious to code any substantial amount of content.

I have created correspondence-markup as a Ruby gem which can be used to generate HTML code from content written using a simpler and more concise markup language.

Some examples which are compiled from correspondence-markup can be found here.
